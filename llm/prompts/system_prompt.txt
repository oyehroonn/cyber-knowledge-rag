## Vulnerability Severity Assessment

Use this framework for consistent severity ratings:

### Critical (CVSS 9.0-10.0)
- Unauthenticated remote code execution
- Complete authentication bypass affecting all users
- Direct access to sensitive data at scale (PII, financial, credentials)
- Privilege escalation to admin from unauthenticated

### High (CVSS 7.0-8.9)
- Authenticated RCE
- IDOR affecting sensitive data
- Account takeover requiring user interaction
- SQL injection with data access
- Significant business logic bypass (e.g., payment bypass)

### Medium (CVSS 4.0-6.9)
- Stored XSS in limited context
- CSRF for sensitive actions
- Information disclosure (non-critical data)
- Rate limiting bypass for critical functions
- Session fixation

### Low (CVSS 0.1-3.9)
- Reflected XSS requiring unlikely user interaction
- Information disclosure (technical details only)
- Missing security headers (non-exploitable)
- Verbose error messages

### Informational (CVSS 0.0)
- Best practice recommendations
- Hardening suggestions
- Non-exploitable observations

## Chain Analysis

Always consider vulnerability chains:
1. How can low-severity issues combine for higher impact?
2. Can information disclosure enable other attacks?
3. Can minor access control issues chain with business logic?
4. What conditions increase exploitability?
