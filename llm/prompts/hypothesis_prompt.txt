## Task: Generate Attack Hypotheses

Analyze the object model, permission matrix, and workflows to identify potential security invariant violations. Focus on:

### 1. Access Control Analysis
- Can User A access User B's resources?
- Can a lower-privileged user access higher-privileged functions?
- Are there horizontal privilege escalation opportunities?
- Are there vertical privilege escalation paths?

### 2. Business Logic Analysis
- Can workflow steps be skipped or reordered?
- Can state transitions be manipulated?
- Are there race conditions in multi-step processes?
- Can pricing, discounts, or limits be bypassed?

### 3. Data Flow Analysis
- Where does sensitive data flow?
- Are there data exposure points?
- Can data be manipulated between steps?
- Are there injection points in data processing?

### 4. Authentication/Session Analysis
- How are sessions managed?
- Are there session fixation opportunities?
- Can authentication be bypassed?
- Are there token manipulation possibilities?

## Hypothesis Generation Guidelines

1. **Be Specific**: Reference exact objects, actions, and parameters
2. **Think Adversarially**: What would a determined attacker try?
3. **Consider Context**: Business impact matters as much as technical severity
4. **Look for Chains**: Single issues that combine for greater impact
5. **Prioritize Wisely**: P1 = likely vulnerable AND high impact

### Priority Definitions
- **P1**: High likelihood + High/Critical impact - Test immediately
- **P2**: Medium likelihood + Medium+ impact OR High likelihood + Low impact - Test soon
- **P3**: Low likelihood OR informational - Test if time permits
